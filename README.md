# Androtone

A cross-platform JUCE 8 synthesizer plugin. The project is declared in `androtone.jucer`
(Projucer). The desktop build is driven by a hand-maintained CMake script that fetches JUCE via
CPM; the Android build uses Projucer's exported Gradle project.

## Targets

| Platform | Format     | Notes                                                                |
| -------- | ---------- | -------------------------------------------------------------------- |
| Desktop  | VST3       | Windows / macOS / Linux (via `Builds/desktop/CMakeLists.txt`)         |
| Desktop  | Standalone | `.exe` / `.app` / binary                                             |
| Android  | Standalone | APK, from the Projucer-exported Gradle project in `Builds/android/`  |

## Prerequisites

- CMake 3.22+
- A C++17 toolchain (MSVC / Clang / GCC)
- Network access on first configure (CPM downloads JUCE)

Android also needs:

- Android Studio + Android NDK
- The desktop CMake configured at least once. The Projucer-exported Android CMake references JUCE
  sources from `Builds/desktop/cmake-build-debug-visual-studio/_deps/juce-src/`, so CPM must have
  populated that path first.

## Build (desktop)

```sh
cmake -S Builds/desktop -B Builds/desktop/cmake-build-release-visual-studio
cmake --build Builds/desktop/cmake-build-release-visual-studio --config Release
```

CLion users can just open `Builds/desktop/CMakeLists.txt`.

Artifacts:

- `Builds/desktop/cmake-build-release-visual-studio/Androtone_artefacts/Release/VST3/Androtone.vst3`
- `Builds/desktop/cmake-build-release-visual-studio/Androtone_artefacts/Release/Standalone/Androtone[.exe|.app]`

## Build (Android)

Open `Builds/android/` in Android Studio, or run `Builds/android/gradlew assembleRelease`. The
Gradle project and `Builds/android/app/CMakeLists.txt` are generated by Projucer — re-save
`androtone.jucer` to regenerate after changing modules, exporter settings, or the file list.

## Notes on JUCE version

`Builds/desktop/CMakeLists.txt` pins JUCE 8.0.12 via CPM. If that tag is not available on
`juce-framework/JUCE`, bump `GIT_TAG` / `VERSION` to the current 8.0.x patch.
