# Claude project notes

## Project shape

Androtone is a JUCE 8.0.12 audio plugin (synth). The project is described in `androtone.jucer`
(Projucer), and there are two independent build paths:

- A hand-maintained CMake build at `Builds/desktop/CMakeLists.txt` for the desktop targets. CPM
  fetches JUCE at configure time into that build tree ‚Äî no vendored sources.
- A Projucer-exported Android Studio / Gradle project under `Builds/android/` for the Android
  Standalone target.

Targets:

- VST3 ‚Äî desktop hosts (Windows / macOS / Linux)
- Standalone ‚Äî desktop
- Standalone ‚Äî Android (Gradle + NDK; `Builds/android/app/CMakeLists.txt` is Projucer-generated)

## Layout

- `androtone.jucer` ‚Äî Projucer project: groups, modules, and the Android export config
- `JuceLibraryCode/` ‚Äî Projucer-generated module shims + `JuceHeader.h` (do not edit)
- `Builds/desktop/CMakeLists.txt` ‚Äî desktop build (CPM bootstrap, JUCE fetch, `juce_add_plugin`)
- `Builds/desktop/cmake-build-*-visual-studio/` ‚Äî CLion-style build trees (also where CPM lands JUCE)
- `Builds/android/` ‚Äî Projucer-exported Gradle project (regenerated by re-saving the `.jucer`)
- `source/` ‚Äî plugin C++
  - `PluginProcessor.{h,cpp}`, `PluginEditor.{h,cpp}`
  - `Sequencer/` ‚Äî step sequencer (e.g. `StepSequencer.h`)
  - `SoundSources/` ‚Äî voices and sounds (e.g. `SineSound.h`, `SineVoice.h`)
- `scripts/` ‚Äî helper scripts (e.g. `build-projucer.ps1` builds Projucer itself from the JUCE
  that CPM fetched into the desktop build tree)
- `docs/` ‚Äî product requirements, design notes, and any Claude design files. **Read everything in
  `docs/` before making non-trivial changes ‚Äî it is the canonical source of intent for this
  project.**

## Build (desktop)

```
cmake -S Builds/desktop -B Builds/desktop/cmake-build-release-visual-studio
cmake --build Builds/desktop/cmake-build-release-visual-studio --config Release
```

CLion users can just open `Builds/desktop/CMakeLists.txt`.

VST3 lands in `Builds/desktop/cmake-build-release-visual-studio/Androtone_artefacts/Release/VST3/`;
the standalone executable in `‚Ä¶/Androtone_artefacts/Release/Standalone/`.

## Build (Android)

Open `Builds/android/` in Android Studio, or run `Builds/android/gradlew assembleRelease`. The
Gradle project and `Builds/android/app/CMakeLists.txt` are generated by Projucer ‚Äî re-save
`androtone.jucer` after changing modules, exporter settings, or the file list.

Note: the Projucer-generated Android CMake currently references JUCE sources via hard-coded paths
inside `Builds/desktop/cmake-build-debug-visual-studio/_deps/juce-src/`. The desktop CMake must
have been configured at least once so CPM has populated that path; re-save the `.jucer` if the
absolute path doesn't match this checkout's location.

## Keeping the project in sync

The desktop CMake is hand-maintained, not exported by Projucer. When adding files under `source/`:

- Add them to `androtone.jucer` (so the Android export picks them up after a re-save).
- Add them to `target_sources(Androtone ‚Ä¶)` in `Builds/desktop/CMakeLists.txt`.

Plugin identity (synth flag, company, plugin/manufacturer codes, bundle ID) is currently set in
the desktop CMake; the Projucer-side identity defaults have not been aligned, so the Android
export will not match the desktop plugin metadata until that's fixed in the `.jucer`.
